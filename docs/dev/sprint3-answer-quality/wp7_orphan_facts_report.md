## 1. Findings

Orphan facts (facts present, documents row missing) in knowledge_base/db/knowledge.db. See WP7 fix proposal. Counts: DOC-000001 = 14 facts / 0 evidence / 202 source_units / 0 pages / 1 expected_points(141 pts); DOC-000008 = 58 facts / 0 evidence / 0 source_units / 0 pages / 0 expected_points / 48 dangling fact_evidence_map entries.

## 2. Root Cause

Leftover from earlier ingestion: documents/pages/evidence rows deleted or never created, but facts survived. DOC-000001 pseudo term_definition facts were generated by post_ingestion_gate term_definition_sync from expected_points noise (term values are full sentences / section headers, not real terms). DOC-000008 facts look like real ISO 14229-7 data but the document was deleted, and 48 fact_evidence_map entries point to evidence_id values that do NOT exist (dangling references).

## 3. Impact / Risk

### 3.1 Reachable by answer pipeline: YES

All 72 orphan facts are indexed in facts_fts (DOC-000001 14/14, DOC-000008 58/58). The recall path (_inject_direct_term_definition_hits query_api.py:775/816, _inject_direct_requirement_hits line 913) filters ONLY by fact_type, NOT by fact_status. So orphan facts ARE recall candidates.

### 3.2 Concrete pollution risk: YES (verified)

Term "抽检方法" has a real definition in DOC-000019 (FACT-025238) AND a duplicate pseudo-term orphan fact in DOC-000001 (FACT-025246, same definition text from expected_points noise). Both are recalled by the term-definition injection, risking citation confusion or duplicate definitions.

### 3.3 Eval impact: NONE currently

DOC-000001 expected_points (141 pts) are all noise (headers/preface) filtered by _is_substantive, producing 0 eval questions. DOC-000008 has no expected_points. Orphans do not currently affect the eval baseline, but are a latent answer-pollution risk.

## 4. Proposed Governance (non-destructive, auditable)

Per Sprint 3 constraints: no direct DB deletion, auditable, must not break source_unit coverage.

1. Mark the 72 orphan facts fact_status -> 'quarantined_orphan' (reversible status change, NOT deletion).
2. Delete the 48 dangling fact_evidence_map rows for DOC-000008 (point to non-existent evidence_id; pure dangling refs, safe to remove).
3. Add fact_status exclusion filter to the recall injection queries in query_api.py so 'quarantined_orphan' facts are never recalled as answer candidates.
4. Quarantine DOC-000001 expected_points (141 noise pts) and 202 source_units (mark status, do not delete).
