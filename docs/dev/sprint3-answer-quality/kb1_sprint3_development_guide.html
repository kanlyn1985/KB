00

给执行 AI 的总指令

Read First
你不是来重写 KB1 的。你是来把真实答案质量提上去的。

当前系统已经完成 Sprint 2：代码已 push，定义查询 xfail 已解除，Ontology 已以 off/shadow/guard 方式安全接入，测试全绿。Sprint 3 的核心是：在不改变评测口径、不让 Ontology 生成事实、不绕过 evidence_judge 的前提下，把跨文档真实 eval baseline 从 0.30 提升到 0.65–0.85。

先诊断，不盲修

  • 先跑并保存 0.30 baseline。
  • 把失败样本按原因分桶。
  • 先修高频、可验证、低风险 bucket。

只做证据内提质

  • 修召回、rerank、answer policy、citation 对齐。
  • 完善 expected_points 绑定。
  • 修 orphan facts 与 source_unit 追踪。

不要走捷径

  • 不得换更宽松 metric。
  • 不得删除困难样本。
  • 不得让 LLM / Ontology 补无证据事实。

02

不可违反的系统约束

Non-negotiables

证据边界

  • evidence_judge 仍然是事实裁决边界。
  • 答案只能来自候选 evidence / fact / context。
  • 证据不足必须降级，不能靠 LLM 或 Ontology 补条文。
  • Ontology 的输出只能是 signal / constraint / warning / validation。

评测边界

  • 主口径仍为 deterministic + token_overlap。
  • 不得切换到 strong-match / LLM judge 来制造更高分。
  • 不得删除困难样本，除非有明确噪声判定与审计记录。
  • 每次提分必须记录 improved / regressed cases。
Sprint 3 的正确姿势

把 0.30 看成真实体检结果，不要嫌难看。现在最有价值的是把失败样本分清楚：到底是召回没拿到证据、rerank 排错、answer policy 表达不覆盖 expected point、citation 错，还是 expected_points 绑定本身有问题。

WP2

答案质量修复 A：定义 / 术语 / 参数类高频失败

目的：优先修复标准文档知识库最常见、最容易验证的 query 类型：定义、术语、参数、条件、测试方法。这类问题通常能在不大改主链路的情况下显著提分。

建议修复方向

  • 沿用 Sprint 2 的 CJK term_definition 直达召回思路。
  • 扩展标准定义类 query pattern：是什么、定义、含义、指什么、缩写。
  • 参数类 query 要保留数值、单位、范围、条件。
  • 测试方法类 query 要保留步骤、判定条件、引用标准章节。

不要做的事

  • 不要靠 prompt 让 LLM 猜定义。
  • 不要把文档标题块当定义证据。
  • 不要为了 token_overlap 堆无证据关键词。
  • 不要绕过 answer_policy 的证据不足降级。
# 建议局部测试
python -m pytest -q tests/test_mcp_server.py
python -m pytest -q tests/test_answer_definition*.py tests/test_query*.py

# 每个修复点后跑一次小样本 eval
eakb eval run-now --profile deterministic --metric token_overlap --case-filter definition,term,parameter
验收标准
定义/术语/参数类失败明显下降；不新增 unsupported claim；citation 不回归。
提交建议
fix(answer): improve evidence-backed definition and parameter coverage

WP4

Citation 与 unsupported claim 安全门禁

目的：eval 提分不能牺牲安全性。所有答案质量提升都必须同时维护 citation correctness 和 unsupported claim 检查。

新增或强化测试

  • 答案每个关键断言能对应 evidence id。
  • citation 不允许只指向文档标题、目录、封面。
  • 证据不足时不输出“看似完整”的标准结论。
  • 多文档查询不串文档。

建议指标

  • citation_correct_rate
  • unsupported_claim_rate
  • title_block_citation_rate
  • answer_coverage_rate
验收标准
eval 提分后 unsupported_claim_rate 不上升；citation_correct_rate 不下降。
提交建议
test(answer): guard citations and unsupported claims during eval uplift
WP5

Ontology 主动召回过滤：只做 Shadow A/B，不直接启用

目的：评估 Ontology 是否能帮助减少噪声召回、关系错配和实体类型混淆。Sprint 3 只允许 shadow A/B，不允许把 Ontology 过滤直接作为生产门禁。

硬边界

Ontology 可以建议“这个候选可能类型不匹配”，但不能删除唯一证据、不能生成事实、不能绕过 evidence_judge。任何主动过滤必须先证明不会降低 recall。

# 推荐实验模式
KB1_ONTOLOGY_MODE=off      # control
KB1_ONTOLOGY_MODE=shadow   # 记录如果过滤会发生什么
KB1_ONTOLOGY_MODE=guard    # 只输出 warning，不改变 answer

# 推荐 A/B 输出
docs/dev/sprint3_ontology_retrieval_ab_report.md
# - off pass_rate
# - shadow projected pass_rate
# - candidates_would_drop
# - evidence_loss_cases
# - false_positive_filter_cases
# - safe_filter_candidates
指标含义Gate
projected_precision_gain如果启用 ontology 过滤，噪声是否下降。越高越好
evidence_loss_rate被过滤掉的候选中是否有真实所需 evidence。必须接近 0
false_positive_filter_casesOntology 误判导致正确证据被标红。必须逐例审计
answer_changed_by_ontologySprint 3 是否直接改答案。必须仍为 false
验收标准
产出 A/B 报告；证明是否值得 Sprint 4 进入 guard retrieval；Sprint 3 不直接生产启用过滤。

建议技术方案：答案质量提升链路

Quality Uplift Architecture

Sprint 3 不建议先动大架构。推荐围绕“失败样本 → 证据链路 → answer coverage”的最短闭环迭代。

eval failed case
  │
  ├─ expected_points
  │
  ├─ retrieval trace
  │    ├─ evidence absent?     → retrieval/query expansion fix
  │    ├─ evidence low-ranked? → rerank/context selection fix
  │    └─ evidence present?    → answer_policy/citation fix
  │
  ├─ answer trace
  │    ├─ missed expected point
  │    ├─ unsupported extra claim
  │    └─ citation mismatch
  │
  └─ repair task
       ├─ unit/regression test
       ├─ local eval
       ├─ full eval
       └─ case-level diff report

推荐新增中间产物

@dataclass
class EvalFailureBucket:
    case_id: str
    doc_id: str
    query_type: str
    bucket: Literal[
      "retrieval_miss",
      "rerank_misorder",
      "answer_undercoverage",
      "citation_mismatch",
      "expected_point_binding",
      "unsupported_claim",
      "timeout"
    ]
    evidence_ids: list[str]
    expected_points: list[str]
    repair_hint: str

推荐追踪指标

eval.pass_rate
eval.pass_rate_by_doc
eval.pass_rate_by_query_type
eval.retrieval_miss_rate
eval.answer_undercoverage_rate
eval.citation_correct_rate
eval.unsupported_claim_rate
eval.title_block_citation_rate
eval.runtime_seconds
