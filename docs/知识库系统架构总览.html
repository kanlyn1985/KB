📁 名称：Athena · 🧩 架构：六层分层 + Nexus 知识编译 · 🔗 GitHub：TheMaxChan/Athena

💡 设计理念

🎯 从 RAG 到知识编译

传统 RAG 返回散落 chunks，需要人工拼凑。Athena 将原始文档编译为结构化知识产物（Artifact），直接输出任务可用的结构化信息。

🔌 Agent 原生接入

通过标准 MCP（Model Context Protocol）暴露能力，多个 AI Agent（OpenClaw、Claude、Codex、Cursor）可统一接入，无需重复集成。

🔄 飞书 - GitHub 双向同步

飞书 Wiki 作为内容管理端，GitHub 作为版本控制和分发端，通过 sync 工具实现双向同步，确保内容一致性。

🏗️ 六层分层架构

从即时上下文到规则引擎，逐层抽象。知识从碎片走向结构化，从确定性检索走向推理增强。

🔧 MCP 工具清单 31 个工具

类别工具功能
搜索检索 kb_search 全文 + 向量 + 图谱三级召回，三级 LLM 路由，支持追问上下文
kb_get 按路径或关键词获取条目完整 Markdown
知识编译 kb_artifact 组件360视图 / ASPICE 设计规范结构化编译输出
kb_answer 结构化工程答案（搜索 + 答案推理）
写入治理 kb_propose 提议新增/更新条目，走评判引擎评估
kb_feedback 提交纠正/补充/质疑/过时反馈
kb_validate 标记验证状态（已验证/待验证/已过时）
kb_review_queue 查看评判引擎待审队列
kb_rollback 回滚到指定快照版本
图谱推理 kb_graph_query 实体/关系/路径/邻域四种查询模式
kb_reasoning_query 依赖分析 / 影响分析 / 路径解释
kb_object_query 工程对象结构化 Profile
规则质量 kb_rules_check 查询 / 校验编码规则（AUTOSAR / MISRA）
kb_doc_quality 评估设计规范文档完整性（5 维度）
kb_health_check 健康巡检：过时文档 / 断裂引用 / 图谱覆盖率
代码生成 kb_codegen 从 Artifact 生成 AUTOSAR C 代码框架（MISRA 合规）
kb_reindex_multimodal 多模态图片重新索引（OCR + DeepSeek 结构化）
辅助 kb_clear_context 清除搜索上下文，重置追问链路
