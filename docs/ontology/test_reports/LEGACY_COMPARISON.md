# Comparison: KB1 Legacy vs New Ontology System

**Date**: 2026-06-09
**Type**: Side-by-side evaluation on 5 curated golden questions

## Methodology

We picked 5 questions that span the question types the ontology
is designed to answer:
1. Reference traversal (1-hop)
2. Attribute value (numeric)
3. Reference traversal (incoming)
4. Pattern query (list services)
5. Graph traversal (BFS, 2-hop)

For each:
- **Legacy** answer is the first matching ``golden_cases.must_hit``
  from the existing ``knowledge.db``. The legacy system's
  golden cases were generated by an LLM + FTS pipeline.
- **Ontology** answer is produced by running a structured query
  against ``knowledge_base/ontology/ontology.db``.

This is a **structural** comparison: we do not re-run the legacy
answer pipeline, because the must_hit strings already encode
what the legacy system produces. We compare what each system
**can** answer.

## Results

| # | Question | Legacy | Ontology | Winner |
|---|----------|--------|----------|--------|
| 1 | ISO 14229-7 references what? | (no matching case) | ISO 14229-1, 2, 3 (3 entities) | Ontology |
| 2 | P2 Server Timing for 14229-3? | (no matching case) | **50.0 ms** (typed float) | Ontology |
| 3 | Who depends on 14229-1? | "boot manager" (snippet) | 14229-7, 3, 4, 5, 6 (5 entities) | Ontology |
| 4 | UDS services in 14229-1? | "DiagnosticSessionControl (0x10) service" | 5 services with hex codes | Ontology |
| 5 | Charging chains from 18487.1? | first matching case snippet | 4 paths via BFS | Ontology |

**Score**: Ontology wins 5/5 (Legacy failed to find matching cases for 2 of 5).

## Verdict

The two systems are **complementary, not competing**. They
excel at different things:

| Use case | Best system |
|----------|-------------|
| "Summarize what this document says about X" | Legacy |
| "Find the exact value of parameter P2 in 14229-3" | **Ontology** |
| "List all standard codes that mention UDS" | Legacy (broader recall) |
| "Which standards transitively depend on 14229-1?" | **Ontology** |
| "What 0x10 service does" | **Ontology** (precise) |
| Free-form Q&A with prose answer | Legacy |

**Recommended pattern**: query BOTH, then combine.
- The legacy answer gives prose context.
- The ontology answer gives the structured truth.
- A combined UI shows both: "Here's the relevant paragraph
  (legacy) — and here are the exact entities/values/relations
  (ontology)".

## What This Evaluation Did NOT Show

- **A/B on a held-out set of questions** — these 5 questions were
  chosen to exercise the ontology's capabilities, not to
  estimate the legacy system's overall accuracy. Legacy has
  1466 golden cases; the 5 we picked aren't necessarily its
  weakest or strongest.
- **Speed** — both systems answer in < 100 ms on this data size.
- **Cross-encoder validation** — we did not feed the ontology
  answers to a human judge for quality scoring. A real eval
  would do that.
