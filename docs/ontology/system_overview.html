KB1 Ontology System (kb1_ontology)

一个与现有 KB1 遗留系统完全隔离的并行本体驱动知识库系统。支持结构化查询（参数、引用、遍历、服务、定义）并自动回退到遗留系统获取散文式上下文。

系统架构

User Question | v [Router] (combined_query.py) | +-- parameter --> Ontology Handler --> Typed Value (50.0 ms) +-- reference --> Ontology Handler --> Structured Set +-- traversal --> Ontology Handler --> BFS Paths +-- service --> Ontology Handler --> Service List +-- definition --> Ontology Handler --> Title String +-- free_form --> Legacy Handler --> Prose Context | v [Combiner] --> 📐 Structured + 📖 Context

三层类层次结构

Meta Layer（元层）

最顶层抽象概念，不可实例化。所有 Domain 类的父类必须是 Meta 层。

Thing InformationEntity PhysicalEntity ProcessEntity ConceptEntity RoleEntity

Domain Layer（领域层）

按工程角色组织，父类必须是 Meta 层。

OBC.Standard OBC.Component OBC.Parameter OBC.Service OBC.Requirement OBC.TestCase Software.ECU Electronics.Circuit Test.Equipment Systems.Process

Instance Layer（实例层）

具体实体，如 ISO 14229-1、GB/T 18487.1、P2_Server_Timing。

设计决策记录 (ADR)

ADR主题决策
0001Meta-Domain 层规则Domain 类的父类必须是 Meta 层；Meta 类的父类也必须是 Meta 层
0002两级规范化canonical_name 使用激进规范化（去空格、去大小写、去年份）；alias 使用轻量规范化（仅空格+大小写）

与遗留系统的集成

隔离原则 (T2)：除 legacy_bridge.py 外，所有新系统模块禁止导入 enterprise_agent_kb。

组合查询流程：

  1. Router 将查询分类为 6 类之一
  2. 如果是结构化查询（parameter/reference/traversal/service/definition），走 Ontology 路径（<100ms）
  3. 同时走 Legacy 路径获取散文上下文（3-30s，LLM 驱动）
  4. Combiner 将两部分结果合并为统一输出
  5. 如果是 free_form，仅走 Legacy 路径

性能对比：Ontology 路径 <100ms vs Legacy 路径 3-30s。
